AGENT_LOG_RAIL — Rail Freight Operations Integrity Module

Market: Global $250-300B. UK £5-8B, ~85M tonnes/yr. Key UK: DB Cargo, Freightliner, GB Railfreight, Colas Rail. ORR regulated. Network: ~10,000 miles, 50+ terminals

1. Mission

Rail freight operations: wagon maintenance compliance, manifest manipulation, intermodal integrity, and network access charge forensics.

2. Detection Architecture

2.1 Wagon Maintenance Compliance

  • W1 — Exam Intervals: TOPS exam record vs. mileage/days. Overdue = safety risk
  • W2 — Brake Test: Brake force records vs. wagon type standards
  • W3 — Wheel/Rail Wear: Flange thickness, tread profile beyond standards

2.2 Load Manifest Integrity

  • L1 — Weight Discrepancy: Declared weight vs. weighbridge at origin/destination
  • L2 — Hazmat Declaration: IMDG/RID class vs. actual cargo. Misdeclaration = safety critical
  • L3 — Loading Pattern: Wagon load distribution anomalies (overhang, focus)

2.3 Intermodal Transfer Integrity

  • I1 — Unit Identification: Container/wagon ID mismatch at transfer point
  • I2 — Transfer Time: Road to rail terminal processing time vs. benchmark
  • I3 — Damage Transfer: Cargo damage origin (road vs. rail vs. ship attribution)

2.4 Network Access Charging

  • N1 — Tonnage Reporting: Declared tonnage for access charges vs. actual
  • N2 — Timetable Adherence: Path usage vs. allocated path. Cancellations and whether used
  • N3 — Possession Manipulation: Engineering possession scheduling vs. actual work performed

3. Proprietary Indicators

Indicator Description Threshold
WAGON_M days_since_last_exam / exam_interval >1.0 = OVERDUE
WEIGHT_D declared_wieght - weighbridge_weight >5% = MISDECLARATION
INTERMODAL_T terminal_processing_time >2x benchmark = INEFFICIENCY
ACCESS_T declared_tonnage - actual_for_access >3% = CHARGE_EVASION
