AGENT_LOG_CONT — Container Shipping & Port Integrity Module

Market: ~200M TEU/yr globally. UK ports handle ~10M TEU. Key UK Ports: Felixstowe, Southampton, London Gateway, Liverpool, Tilbury

1. Mission

Container shipping integrity: VGM manipulation, container condition forensics, terminal data integrity, and demurrage/detention fraud detection.

2. Detection Architecture

2.1 VGM/SOLAS Compliance

  • V1 — Weight Discrepancy: Declared VGM vs. actual weight at terminal. >2T gap = safety/ fraud risk
  • V2 — Shipper Pattern: Same shipper's declarations consistently below threshold
  • V3 — Overweight: Container weight > maximum gross weight. Structural risk

2.2 Container Condition Forensics

  • C1 — Reefer Temp: Temperature log vs. setpoint. Breach detection
  • C2 — Shock/Vibration: Impact recorder data vs. handling event claims
  • C3 — Tamper Detection: Seal number vs. BOL. Missing/broken = theft indicator

2.3 Terminal Operations Integrity

  • T1 — Gate Processing: Truck turn time anomalies (waiting, staging)
  • T2 — Crane Productivity: Moves per hour vs. industry benchmark
  • T3 — Yard Position: Container location vs. system record matching

2.4 Demurrage/Detention Fraud

  • D1 — Free Time Calculation: Claimed vs. contractual free days
  • D2 — Gate-In/Gate-Out: Actual container availability vs. demurrage clock
  • D3 — Equipment Return: Chassis/reefer return time disputes

3. Proprietary Indicators

Indicator Description Threshold
VGM_D declared_weight - terminal_weight >2T = FRAUD_RISK
REEFER_B time_outside_temp_range >30min = BREACH
SEAL_I bol_seal - actual_seal mismatch = TAMPERING
